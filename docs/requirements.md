# 要件定義書

**プロジェクト名:** rpi-voice-agent  
**作成日:** 2026-06-11  
**バージョン:** 1.0.0

-----

## 1. プロジェクト概要

Raspberry Pi 5 上で動作する、完全ローカルの音声対話エージェントシステムを構築する。クラウドサービスへの依存を排除し、ウェイクワードによるハンズフリー起動から音声応答までの一連のパイプラインをオンプレミスで完結させる。

-----

## 2. 背景と目的

### 背景

- 持ち運べる小型デバイスで常に手元にあるパーソナルアシスタントを作ってみたい
- 外出先や障害時にインターネット接続に依存するシステムは信頼性が低い
- LLM の小型化（1B〜3B パラメーター）により、Raspberry Pi クラスのハードウェアでも実用的な推論が可能になった

### 目的

- **プライバシー保護:** 音声・会話データをすべてローカルで処理し、外部に送信しない
- **コスト削減:** 毎月のクラウド API 費用をゼロにする
- **オフライン動作:** インターネット接続なしで動作する
- **再現性:** GitHub のリポジトリから認証情報以外は自動でゼロ構築できる

## 4. 機能要件

### 4.1 音声入力

|ID  |要件                             |
|----|-------------------------------|
|F-01|ウェイクワードを発話することでシステムが起動すること     |
|F-02|ウェイクワードは設定ファイルで変更可能であること       |
|F-03|カスタムウェイクワードモデル（.tflite）を追加できること|
|F-04|マイクからの音声をリアルタイムでテキストに変換できること   |
|F-05|日本語の音声認識に対応すること                |

### 4.2 エージェント処理

|ID  |要件                              |
|----|--------------------------------|
|F-06|テキスト化されたコマンドをローカル LLM で処理できること  |
|F-07|LLM モデルは設定で切り替えられること            |
|F-08|エージェントがセッションをまたいで記憶を保持できること     |
|F-09|スキル（タスク定義）を追加・拡張できること           |
|F-10|Home Assistant 経由で外部サービスと連携できること|

### 4.3 音声出力

|ID  |要件                                  |
|----|------------------------------------|
|F-11|エージェントの応答をテキストから音声に変換して出力すること       |
|F-12|TTS はローカルで動作し、クラウド API を使わないこと|
|F-13|日本語 TTS に対応すること                     |

### 4.4 システム管理

|ID  |要件                                        |
|----|------------------------------------------|
|F-14|ワンコマンドで全サービスをセットアップできること                  |
|F-15|認証情報（API キー等）以外は Git リポジトリで管理できること        |
|F-16|各サービスの起動・停止・再起動が Docker Compose コマンドで行えること|
|F-17|サービスの異常終了時に自動で再起動すること                     |
|F-18|Ollama モデルの追加がスクリプト一つで行えること               |
|F-19|セットアップ処理（HA 初期化・Wyoming 統合・Assist パイプライン作成等）は Docker の `setup` ワンショットサービスに集約し、ホスト側での処理を apt-get・Docker インストール・コンテナ起動前に必要なファイル生成のみに限定すること。将来の初期化処理追加もこのサービスに収容できる設計とする|

## 5. 非機能要件

### 5.1 性能

|ID  |要件                                            |
|----|----------------------------------------------|
|N-01|ウェイクワード検出のレイテンシが 500ms 以内であること                |
|N-02|LLM の応答生成が開始されるまで 30 秒以内であること（Pi 5 / 3B モデル想定）|
|N-03|システム全体のメモリ使用量が 7GB 以内に収まること（8GB RAM モデル前提）    |

### 5.2 信頼性・可用性

|ID  |要件                     |
|----|-----------------------|
|N-04|電源再投入後、全サービスが自動起動すること  |
|N-05|個別サービスの障害が他サービスに波及しないこと|
|N-06|ヘルスチェックが各サービスに設定されていること|

### 5.3 セキュリティ

|ID  |要件                                    |
|----|--------------------------------------|
|N-07|認証情報（API キー）を Git リポジトリにコミットしないこと     |
|N-08|Hermes Agent の API エンドポイントに認証キーを設定すること|
|N-09|Ollama は外部ネットワークに公開しないこと（LAN 内のみ）     |

### 5.4 保守性・移植性

|ID  |要件                                             |
|----|-----------------------------------------------|
|N-10|Raspberry Pi OS Bookworm 64-bit (arm64) で動作すること|
|N-11|Docker Compose でサービスを定義し、環境差異を吸収すること           |
|N-12|README とドキュメントで再セットアップ手順が完結すること                |

## 6. 制約・前提条件

- **ハードウェア:** Raspberry Pi 5（8GB RAM 推奨）
- **OS:** Raspberry Pi OS Bookworm 64-bit
- **GPU アクセラレーション:** Pi 5 の VideoCore GPU は Ollama 非対応のため CPU のみ
- **推奨モデルサイズ:** 3B パラメーター以下（メモリ・速度のトレードオフ）
- **初回インターネット接続:** セットアップ時に Docker イメージと LLM モデルのダウンロードが必要
- **Home Assistant のオンボーディング:** `setup` コンテナが自動実行（手動操作不要）
- **Mac 開発検証:** Apple Silicon Mac + Docker Desktop での検証に対応。`bash setup.sh --mac` で起動。PulseAudio TCP 経由で音声を接続。Pi 5 本番環境とは音声経路が異なる点に注意
